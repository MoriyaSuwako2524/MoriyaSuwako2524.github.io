Minecraft Login Problems Note

English

I encountered a login issue while playing GTNH (a Minecraft modpack). Although I successfully logged into my official account through the PCL launcher, I was unable to join any servers with authentication enabled — the game consistently displayed the message “Login failed: Authentication servers are down for maintenance.”

![Figure1](/pngs/login fail.png)

Most online solutions suggested modifying the hosts file, refreshing the DNS cache, re-logging into the account, or enabling multiplayer mode in the Microsoft account settings. However, none of these methods worked for me. Even after switching to a different launcher, the problem persisted.

Eventually, I discovered that the issue was caused by Java. PCL had automatically downloaded Java 8 for GTNH, but for some reason it failed to connect properly. In this case, the launcher displayed “OAuth: Offline” in the upper left corner. Even after the browser reported a successful login, the status remained offline.

After switching to Java 25, the issue was resolved. The upper-left indicator now correctly shows “OAuth: Logged in.”

![Figure2](/pngs/login suc.png)

中文

我在登录 GTNH（一个 Minecraft 整合包）时遇到了一些问题。虽然我已经通过 PCL 启动器成功登录了正版账号，但在尝试加入任何开启正版验证的服务器时，始终提示：“登入失败：身份验证目前正在停机维护。”

![Figure1](/pngs/login fail.png)

网上常见的解决方法包括修改 Hosts 文件、刷新 DNS、重新登录账号，以及在XBOX&微软账户中启用多人模式。然而，我在逐一尝试后依然无法解决问题。即使更换启动器，情况也没有改善。

最终我发现问题出在 Java 上。PCL 为 GTNH 自动下载了 Java 8，但由于某些原因，它无法正确建立身份验证连接。此时启动器左上角会显示 “OAuth：离线”。即便浏览器中提示登录成功，状态依然保持离线。

当我将 Java 版本更换为 Java 25 后，问题立刻得到解决。此时左上角显示为 “OAuth：登录”，可以正常进入服务器。

![Figure2](/pngs/login suc.png)
